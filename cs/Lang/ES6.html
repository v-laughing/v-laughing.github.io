解构赋值 ​

解构，将数据拆分为更小部分的过程。 规则：当等号右边的值不是对象或数组，就尝试将其转为对象。

对象解构 ​

找到同名属性，然后再赋给对应的变量，即按变量名解构，但可对其设置默认值及重命名。

js
let {x: x1 = 3, y = 2} = {x: 5};
x1 //5
y //2

可以嵌套解构。

js
let node = {
	loc: { x: 1}
};
let { loc: {x} } = node;
x //1

数组解构 ​

按顺序解构，可以省略，设置默认值，也可嵌套。

js
let [, b, [c] ] = [1, 2, [3, 4]]; // b:2, c:3

不定元素：放在末尾

js
let [a, ...rest] = [1, 2, 3];

Set与Map ​

ES5中模拟下面有个有点模棱两可。

  • Set：检查对象中是否存在某个键名。（in运算符也会检索对象的原型。）
  • Map：获取已存的信息。

集合中元素的相等比较使用SameValueZero操作，基本相当于===。有几个特殊情况：

  • NaN相等。
  • +0与-0等同于0。

Map ​

储存着键值对的有序对象。键值可以是任意类型。

js
let m = new Map([ ['a', 1] ]);
m.set('b', 2);
m.has('a');  // true
m.forEach((value, key, map) => {});
m.entries() //默认迭代器

与Object性能对比：

  • map内存占用较低。大量插入、删除操作下，map更好。
  • 大量查找下，Object更好点。

WeakMap ​

集合中的键必须是一个对象（可非引用）。

其中的key是一种弱引用，不会阻止垃圾回收。通常key应为对象的引用。key不可迭代。

js
const a = {a:1}
const wm = new WeakMap()
wm.set({}, 'null')
wm.set(a, 'null')
wm.has({}) // false
wm.has(a) // true

因为WeakMap实例不会妨碍垃圾回收，可用于保存关联数据，例如DOM节点的状态。当节点从DOM树中移除后，可顺利地释放其内存。

js
const el = document.querySelector('#title')
wm.set(el, {disable: true})

Set ​

有序且没有重复值的列表。元素可为任意类型。

js
let s = new Set([1, 2]); //任意可迭代对象
s.add(3);
s.has(1); //true
s.values(); //默认迭代器

forEach()方法的回调函数前两个参数的值是一样的。

WeakSet ​

同样，集合中的值必须是一个对象（可非引用）。值是一种弱引用。

函数 ​

默认参数 ​

  • 可以放在任意位置。非末尾位置时，实参为undefined时使用默认值。
  • arguments对象不带有默认参数。
  • 默认参数是在函数调用时求值。

不定参数 ​

处理无命名参数，即多出来的实参。

js
function fn1(first, ...args) {
    console.log(first);  // 10
    console.log(args);  // 一个数组：[20, 30]
}

fn1(10, 20, 30);
  • 只能有一个不定参数，且需置于末尾。
  • 函数的length属性统计命名参数的数量，不包含不定参数。
  • arguments包含所有传入的参数。

name属性 ​

以字符串的形式返回函数的名字，便于调试使用。

  • 函数声明及表达式定义返回预期的名字。
  • 访问器属性函数getter和setter的名字带有前缀get或set。
  • bind()创建的函数的名字带有"bound"前缀；Function构造函数创建的函数的名称带有前缀"anonymous"。

箭头函数 ​

  • 没有this、super、arguments和new.target绑定，其值由外围最近一层非箭头函数决定。
  • 不能通过new关键字调用， 没有原型。
  • 不能通过call()、apply()或bind()方法来改变其内this的值。
js
let y = x => x * x;
let z = (x, y) => {return x * y;};

其他特性 ​

  • 增强了Function构造函数的功能，支持在创建函数时定义默认参数和不定参数。

迭代器与生成器 ​

迭代器：一种特殊对象，为迭代过程设计的专有接口。迭代器有一个next()方法，每次调用都返回一个结果对象{value, done}。 生成器：返回迭代器的函数。

  • 迭代器调用next()后，程序yield 1并终止（左侧未执行），直到下次next()。
  • 迭代器next(2)传值，值赋给上个yield式子的左侧变量，然后yield 2。
  • return 3语句可使迭代器提前返回，但返回值不作为迭代器的生成值。
js
function *generator(){
	let a = yield 1; //
	console.log(a);
	yield 2;
	return 3;
}
let iterator = generator();
iterator.next(); // {value: 1, done: false}
iterator.next(2); // a:2, 
iterator.next(); // {value: 3, done: true}

委托生成器 ​

从生成器中产生值，并以此构建生成器，类似与python的yield from。

js
function *generator2(){
	yield *generator();
}

异步编程 ​

Promise ​

对异步操作的一层封装，当异步操作完成时，改变其状态，执行引入的回调函数。

Promise有三种状态：pending、fulfilled和rejected，且：

  • 状态不被外界改变。
  • 状态变化只有可能：pending=>fulfilled和pending=>rejected。
  • 一旦状态发送改变，就不会再变。

具体流程：创建Promise时，其执行器会立刻执行，此时Promise状态为pedding。异步操作结束后，Promise通过resolve, reject函数调用进入fulfilled或rejected状态，之后将then()方法或catch()方法传入的函数推入任务队列。

thenable对象：实现了then()方法的对象。 非Promise的Thenable对象：拥有then(resolve, reject)方法的普通对象。

  • 如果执行器内部抛出一个错误，则Promise的拒绝处理程序就会被调用。
js
let a = new Promise((resolve, reject)=>{
	console.log('hello');
	resolve(3);
});
a.then((res)=>{
	console.log(res);
});
// hello, 3

创建已处理的Promise 可以接受非Promise的Thenable对象作为参数，放回一个Promise对象。

js
let a = Promise.resolve(42); //完成态
let b = Promise.reject(137); // 拒绝态

串联Promise 调用then()或catch()方法时创建并返回了另一个Promise。串联的Promise被依次执行。返回值可以是任意类型，若非Promise类型会将其转为Promise类型。

Promise链可以使得异常在链中依次向下传递，因此务必在链末尾留有一个拒绝处理程序，以确保能够正确处理所有可能发生的错误。

其他方法 ​

promise.all()：并行执行promises，返回Promise对象。当全部接受时，其状态为接受。当第一个拒绝时，其为拒绝。

Promise.race()：返回一组集合中最先解决或拒绝的期约的镜像。

继承Promise ​

重写then，catch名字：

js
class MyPromise extends Promise{
	success(resolve, reject){
		this.then(resolve, reject);
 	}
	failed(reject){
		this.catch(reject);
 	}
}

基本实现 ​

回调封装与链式调用：

js
function MyPromise(fn){
    var self = this;
    self.status = 'pending';
    self.onResolvedCallback = [];
    self.data = null;
    function resolve(data){
        if(self.status == 'pending'){
            self.status = 'resolved';
            self.data = data;
            self.onResolvedCallback.forEach(callback => callback(data));
        }
    }
    fn(resolve);
}
MyPromise.prototype.then = function(onResolved){
    var self = this;    
	return new Promise((resolve)=>{
		self.status == 'pending' ?
		self.onResolvedCallback.push((value)=>{
			resolve(onResolved(value));
		}) : resolve(onResolved(self.data));
	});
}

async ​

[ES2017]

async函数返回一个 Promise 对象，内部return语句返回值成为then方法回调函数的参数。

  • 可以用在任意函数上，包括匿名函数。
  • 内部产生错误，其返回的Promise对象状态变为reject。

捕获内部错误本质通过try,catch实现，因而无法捕获异步代码，如Promise.reject(0)。可以再前面加await命令捕获异步错误。

await命令 接 Promise 对象，阻塞异步函数的执行并等待该对象resolve的结果。

当await右边的值可用时，JS引擎向消息队列中推送一个恢复该异步函数执行的任务。

  • 可接常量或thenable对象。
  • 只能用在async函数中。
js
async function foo() {
	print(1);
	await 2;
	print(3);
}
foo();
print(2); // 1 2 3

并发执行写法

js
let a = afun1();
let b = afun2();
let res_a = await a;
let res_b = await b;
// 等价于let [res_a, res_b] = await new Promise.all([afun1(), afun2()]);

Proxy & Reflect ​

JavaScript中一切皆为对象。对象的基本操作行为由其内部方法所定义，采用[[xxx]] 来代表内部方法。通过内部方法可以区分对象，例如函数对象会部署内部方法 [[Call]]，而普通对象则不会。

每个内部方法都有一个对应的自定义方法名称，允许我们重新定义对象的基本操作。

Proxy ​

Proxy是一种对对象的封装，在内部可以对该对象进行操作。同时，可以重新定义对Proxy的内部方法，从而间接实现对代理对象的操作拦截。

  • get：拦截对象属性的读取 比如p.age，p[age];
  • set： 拦截对象属性的设置 比如 p.age=18,返回一个布尔值
  • has：拦截 propKey in proxy 的操作，返回一个布尔值。

Reflect ​

对象操作的API，包含对象所有的内部方法调用，使得对对象的基本操作都变成函数行为。

Reflect 对象的方法与 Proxy 对象的方法一一对应，使得Proxy可以通过 Reflect 上获取对象的默认行为。

Reflect也可以修改对象的默认行为，使得满足需要。如下的代码中，p.bar时this指向的是obj而不是p，此时的Proxy行为并没有很好的代理。使用Reflect可以指定对象操作时的上下文，从而使得this指向p。

js
const obj = {
  foo: 1,
  get bar(){
	return this.foo
  }
}

const p = new Proxy(obj, {
  get(target, key, receiver){
	return target[key]
	// return Reflect.get(target, key, receiver)
  }
})
