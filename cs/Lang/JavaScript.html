语言概念 ​

内置对象：指在程序执行前存在全局作用域的值、函数和构造函数、对象等。如NaN、undefined，parseInt()、Date、Object 、Math对象。

执行上下文 ​

作用域 ​

每个作用域可以视为一个执行上下文（执行环境），且有一个与之关联的变量对象，上下文中定义的所有变量和函数都保存在这个对象中。

  • 每个函数都有自己的执行环境。
  • 作用域链的用途，是保证对执行环境有权访问的所有变量和函数的有序访问。标识符解析是沿着作用域链逐级地搜索标识符的过程。
  • 没有块级作用域。

执行上下文包含三个属性：变量对象、作用域、this。

执行上下文的生命周期分为两个阶段：

  • 创建： 创建变量对象，建立作用域链，以及确定this的指向。
  • 执行：执行代码，为变量赋值，即将变量对象的属性赋值。

变量对象创建过程：

  1. 建立arguments对象。
  2. 确定函数声明：以函数名作为属性，属性值指向该函数的引用。
  3. 确定变量声明：以变量名作为属性，属性值为undefined。如果该变量名的属性已经存在，不会进行重置undefined，因为可以重复声明。

函数执行上下文中的处于执行阶段的变量对象，一般称为活动对象。 全局上下文的变量对象为window。

this ​

表示当前执行上下文。 一般用于函数内部，表示实际执行函数的对象。

  • 调用函数时，this 绑定到全局对象。
  • 调用方法时，this 绑定到所属对象。
  • 调用构造函数时，this 绑定到创建后的对象。

改变this ​

js
(obj.foo = obj.foo)(); //window
(obj.foo , obj.foo)(); //window

通过apply()、call()、bind()可以指定函数执行的作用域，继而改变this值。这三个方法是所有函数的属性。

前两个会执行函数，后一个返回新的函数：

js
var person = {  
	name: ""  
};  
function setName(firstName,lastName){  
	this.name = firstName + lastName;  
}
setName.apply(person, ["xiao","ming"]);  
// 等价于 setName.call(person,"xiao","ming");
let pSetName = setName.bind(person);
pSetName();

声明 ​

变量提升 ​

函数或var声明的变量，其声明会置于当前作用域顶部，函数声明在先。可重复声明。

js
console.log(a); // undefined
var a = 'a';

原因：JS解析过程中，创建一个代表当前作用域的变量对象，其包含所有声明的函数及其形参和变量。JS执行时通过该对象寻找变量。

下面的例子中，getName声明两次，执行前被重新赋值。

js
var getName = function () { alert (4);};
function getName() { alert (5);} 
getName(); // 4

块级声明 ​

[ES6] 通过let或const，把变量的作用域限制在当前代码块{}中。与var声明变量的区别：

  • 禁止重声明
  • 没有变量提升：临时死区
  • 全局作用域下声明的变量，不会作为全局对象的属性。

const：声明变量后，不允许修改绑定，但允许修改值。

严格模式 ​

ES5通过use strict指定JS的严格模式。为了：

  • 消除 Javascript 语法的一些不合理、不严谨之处，减少一些怪异行为;
  • 消除代码运行的一些不安全之处，保证代码运行的安全；
  • 提高编译器效率，增加运行速度；
  • 为未来新版本的 Javascript 做好铺垫。

主要特点：

  • 禁止使用 with 语句。
  • 禁止 this 关键字指向全局对象。
  • 对象不能有重名的属性。
  • 变量必须被声明。
  • eval() 会为其中声明的变量等创建新的作用域。
  • Arguments 对象只持有静态的参数复制，而非对参数的引用。
  • 对 call stack 的检查被禁用，所以无法使用 arguments.caller 和 arguments.callee，它们会返回 TypeError。

数据类型 ​

基本数据类型：占据空间小、大小固定，被频繁使用的数据，一般在栈中存储。

基本类型不是对象

主要有：

undefined：声明但未初始化的变量的值。

  • 可通过void 0 来获得 undefined。
  • undefined 不是一个保留字。
  • 以下情况会得到该值：变量未声明、无return函数的返回值、未传参。

Null：空对象指针。

Boolean：该类型只有两个字面值：true和false。

假值：undefined、null、false、+0、-0、NaN、""

Number：包括整数及浮点数。

String: 不可变的。

Symbol[ES6]：独一无二且不可变的值。可以防止属性名冲突。

引用类型：占据空间大、大小不固定， 在堆中存储数据，栈中存储指针。如Object 类型，包括Array、Date等。

Number ​

安全整数：转化为二进制存储的时不会出现精度丢失的整数。

以 64 位的格式存储，即"双精度浮点数"，安全整数范围为[1−253,253−1]。 为了节省空间，JS会保证值相等的情况下，尽可能将浮点数转为整数。

js
let n = 1.3e4; //13000
2 === 2.0 // true

特殊值 ​

Infinity：当某次运算得到的值超过[Number.MIN_VALUE, Number.MAX_VALUE]时，会得到该值。该值不能参与运算。

NaN：非数字。数字运算的异常结果。与任何值都不相等，包括NaN本身。

数值转换 ​

Number()：null或“”时返回0；转换字符串时，全部字符需都可转换，否则返回NaN。

+、-操作符会先调用这个，再转换。

parseInt()：转换其中满足数字模式的前部分；否则返回NaN。

parseFloat()：与parseInt()相比，会忽略前导的零。

js
Number("12a"); // NaN
parseInt("12a"); // 12
parseFloat("0xa"); // 0

----[ES6]----

数值拓展 ​

Number.isFinite(), Number.isNaN()：与全局函数相比，其不会事先用Number()进行转换。

Number.EPSILON：JS能够表示的最小精度。可以用来设置“能够接受的误差范围“。

Number.isSafeInteger()：判断是否在有效整数精度范围内。

精度计算 ​

toPrecision：处理精度，精度是从左至右第一个不为 0 的数开始数起。

toFixed： 对小数点后指定位数取整，从小数点开始数起。

Math.round： 是将一个数字四舍五入到一个整数。

----[ES2020]----

BigInt ​

一种新的数据类型，可以精确表示任何位数的整数。

  • 不能使用正号。
  • 不能与NUmber类型直接参与运算，除非不会存在损失精度。
js
let a = 123n; // 后面加n
1 + a; // error
1 < a; // true

字符串 ​

编码 ​

JS的字符串里的字符有两种，一种是由一个编码单元16位表示的BMP字符，另一种是由两个编码单元32位表示的辅助平面字符。 在ES5中，所有字符串的操作都基于16位编码单元。

[ES6] 新增一些支持通过字符来操作字符串的方法。

js
// 根据码点生成字符串
let a = String.fromCodePoint(134071);
a.length  // 2

normalize()：在对比字符串之前，需把它们标准化为同一种形式。

类型转换 ​

通过String()可以将其他值转为字符串。

  • 如果值有toString()方法，则返回相应的结果。
  • 如果值是null或undefined，则返回"null"或"undefined"。
js
String([1,2]); // "1,2"
String(false); // "false"
String({}); // '[object Object]'

正则表达式 ​

js
let re = /[ab]c/gi;
let re2 = new RegExp('[ab]a', 'g');
let re3 = new RegExp(/[ab]c/g); // [ES6]

匹配模式主要有：

  • g：全局模式，在字符串中匹配所有符合的子串，而匹配第一个。
  • i：不区分大小写。
  • m：多行模式。
  • [ES6] u：以字符模式进行匹配，而不是以编码单元模式。
  • [ES6] y：从指定位置匹配，如果没能成功匹配，则停止继续匹配。

主要方法：

  • exec()：找到匹配的捕获组。也可以通过RegExp.$1得到第一个捕获组。
  • test()：判断是否匹配。

数组 ​

大小动态调整，每一项可以保存任何类型的数据。

js
let a = [1, 2];
let b = Array(3); // 数量为3
let c = Array(1, 3); // [1, 3]

属性与方法 ​

  • length属性可以更改，进而影响原数组。
  • 栈方法：push、pop；队列方法：shift、unshift;
  • sort方法会默认对数据进行toString()后进行比较，可以通过指定比较函数覆盖默认。
  • slice从原数组中切出一个新数组。
  • splice可以从原数组中插入或删除数据。
js
let is = Array.isArray(a); // true
let str = a.join('_');  // 1_2
let every = [0, 2, 2].every(); //false
let index = a.indexOf(2); // 1

[ES6]

  • Array.of()：创建一个包含所有参数的数组。
  • Array.from()： 从可迭代对象或类数组对象产生数组。
  • find()和findIndex()：按条件（以回调函数形式）查找第一个匹配的。
  • fill(): 填充。
js
let a = Array.of(1, 2, 3); // [1, 2, 3]
let b = Array.from(a, v => v * 2);  // [2, 4, 6]

类数组对象 ​

拥有 length 属性和若干索引属性的对象，如 arguments 和 DOM 方法的返回结果，甚至函数。

类数组对象一般通过 call、apply来调用数组的方法。

定型数组 ​

通过缓存区，视图构建的用于处理数值类型数据的专用数组，不是Array的继承。

  • 共有8中数据类型。
  • length只读。
  • 拥有大部分Array的方法。
  • 会检查输入数据类型的合法性，0被用于代替所有非法值。
js
let ints = new Int16Array(3); 
let ints2 = new Int16Array([1,2,3]);

Symbol ​

[ES6] 表示一种基本类型，不是对象。

js
let sb1 = Symbol();
let sb2 = Symbol("name"); //参数仅仅作为辨认的标识，一个描述

[ES2019]可以通过description属性获取其描述参数值。

一般用来定义常量，消除魔术常量以及作为特定的属性（以防止属性重叠）。作为属性时，注意与字符串区分，不能用点运算符。

js
let a = {
	[sb2]: "sb2"
};
a[sb1] = "symbol";
a[sb1] // symbol

常用的属性获取方法无法得到Symbol属性，如for in、for of，Object.keys()等，需通过Object.getOwnPropertySymbols方法获取。

可以利用这点，定义非私有的内部方法或属性。

登记机制 ​

登记一个symbol，之后可通过参数获取该symbol。Symbol.for()可以获取已登记的值或新建一个。

js
let a = Symbol.for("foo");
let b = Symbol.for("foo"); //a===b
Symbol.keyFor(b); // "foo"

登记是全局性质的，可以以此在不同的 iframe 或 serviceworker 中取到同一个值。

内置Symbol ​

ES6内置了一些Symbol值，作为对象的属性，指向特定的方法。当对对象执行某些操作符或方法时，便会调用这些Symbol定义的方法。

这些操作符或方法有：instanceof、concat()、match、replace、search、split、iterator、with等等。

js
let a = {
	[Symbol.hasInstance](ins){return true;}
}
"" instanceof a; //true

Symbol.toStringTag：定制对象的[[Class]]值。

类型转换 ​

可以强制转换成字符串或布尔类型，但不能转为数字类型，否则异常出错。

类型检测 ​

typeof ​

检测基本数据类型和引用类型，前者返回具体的类型，如underfined、string等；后者统一返回object。

一般用于检测基本类型。

js
let func = ()=>{};
typeof null; // 'object'
typeof func; // 'function'
typeof a; // undeclared变量，返回"undefined"

instanceof ​

检测具体是哪种引用类型，其右边必须是对象或构造函数。 本质通过判断左边对象的隐式原型能否到达右边构造函数的显示原型。

js
[1] instanceof Array; //true

Object.prototype.toString ​

返回[object className]，其中className具体值由对象的内部属性[[Class]]指定。

js
let b = "string", c = /regex/i; 
Object.prototype.toString.call(b); // [object String]
Object.prototype.toString.call(c); // [object RegExp]

js
function getType(value) {
	if (value === null) return value + "";
 	if (typeof value !== "object") return typeof value;
  	let valueClass = Object.prototype.toString.call(value);
	return valueClass.slice(1,-1).split(" ")[1];

以字符串的形式返回变量的类型。null -> object 。

操作符与语句 ​

逻辑运算 ​

!返回布尔值，&&和||返回符合的操作数。后者经常用来做短路运算。

js
let a = undefined && [1,2]; // [1,2]

!运算符行为：对[[#数据类型|falsy值]]返回true，否则返回false。

相等判断 ​

==：会进行强制类型转化：

  • boolean，number，string时：bool->number，string->number。
  • 仅其中之一是对象，先调用其valueOf()取得比较值；两者为对象，比较引用。
  • undefined与null不转换，二者相等。
js
undefined == null; //true
NaN == NaN; // false

===：不会做强制类型转换。

Object.is()：比===更严格，准确的值比较。

js
Object.is(+0, -0); // false
Object.is(NaN, NaN); // true

基本类型比较值，对象比较引用。

+ ​

加号：

  1. 如果有对象，将其转为基本类型。
  2. 如果有字符串，将另一个转为字符串。
  3. 将操作数转成数字。

一元： 将操作数转换成数字并返回。

展开符 ​

[ES6] 将任意可迭代对象展开，用于函数参数传递，可迭代对象转数组，数组拷贝赋值等等。

js
let s = new Set([1, 2, 3]);
let a = [...s]; //转换
let b = [...a]  // 拷贝
let f = (x, y, z) => {};
f(...b); //函数传递

遍历 ​

for in: 用来枚举对象的可枚举属性，包括其原型对象属性；属性顺序是不确定的。

js
for(prop in {'a':1, 'b': 2}){
	console.log(prop); // a, b
}

Object.keys()：自身的可枚举属性。对应的是Object.values()。

Object.getOwnPropertyNames()：自身的所有属性（除了Symbol）。

[ES6]Reflect.ownKeys(obj)：自身的所有属性。

[ES6]for of: 遍历任意可迭代对象。

js
for(prop of [1, 2]){
	console.log(prop); // 1,2
}

with ​

将代码的作用域设置到一个特定的对象中。

js
with(user){
	console.log(name); //name is user.name
}

大量使用with语句会导致性能下降；严格模式下不允许使用with语句。

switch ​

switch语句在比较值时使用的是全等操作符，因此不会发生类型转换。

其他 ​

() ​

语句(statement)不会触发求值操作。

js

(x) = 2; // x:2
(obj.foo)(); // this 为 obj

对象 ​

无序属性的集合。键在底层都是字符串。

引用类型的关系 ​

对象的值保存在堆内存中，而对象的引用（即变量）保存在栈内存中。

属性及其特性 ​

属性可分为两类：数据属性和访问器属性。 属性的特性为描述属性的内部特征，用[[Enumerable]]表示。

  • 数据属性：可直接定义值，通过以下特性描述其行为。
  • Configurable：能否通过delete删除属性从而重新定义属性，能否修改属性的特性，或者能否把属性修改为访问器属性。默认值为true。
  • Enumerable：能否通过for-in循环返回属性。默认值为true。
  • Writable：能否修改属性的值。默认值为true。
  • Value：属性的数据值。读写实际通过该值。默认值为undefined。
  • 访问器属性：通过getter和setter函数描述值的行为。
  • Get：在读取属性时调用的函数。默认值为undefined。
  • Set：在写入属性时调用的函数。默认值为undefined。 Configurable与Enumerable同上。 只指定getter意味着属性是不能写，尝试写入属性会被忽略。

属性访问 ​

  • hasOwnProperty() 可以检测一个属性是存在于实例中，还是存在于原型中。
  • in操作符会在通过对象能够访问给定属性时返回true。
  • Object.keys(oj) 方法以数组方式返回对象oj所有可枚举的实例属性。

判断一个属性是否存在于原型中：

js
!object.hasOwnProperty(name) && (name in object);

特性访问 ​

js
var book = {
	name: 'js'
};

Object.defineProperty(book, "name", {
	writable: false
});

Object.defineProperties(book, {
	version: {
		value: "1.0"
	}
})

var descriptor = Object.getOwnPropertyDescriptor(book, "name");

  • 对象属性可重复，取最后一个。
  • 属性枚举顺序：数字升序排序、字符串加入顺序、symbol键加入顺序。

可计算属性：在对象字面量中使用方括号表示，其值最终为方括号内求值后的字符串。

js

let a = {

	getName(){},  // 方法简写

对象创建 ​

基本方式 ​

通过字常量创建，本质和通过构造函数的方式一样。

js
let a = {n:1}; 
// 等价于 let a = new Object(); a.n = 1;

结合[[#原型与原型链]]的知识，new操作符实际等于：

js
function create(fn, ...args) {
  let obj = {}
  Object.setPrototypeOf(obj, fn.prototype)
  let result = fn.apply(obj, args)
  return result instanceof Object ? result : obj
}

Object.create：创建指定原型的对象。

js
var a2 = Object.create(a);
a2.__proto__ === a; // true

Object构造函数或对象字面量都可以用来创建单个对象。但使用同一个接口创建很多对象，会产生大量的重复代码。

工厂模式 ​

用函数来创建对象。

js
function createObject(name) {
	var o = new Object();
	o.name = name;
	return o;
}
var o1 = createObject('zhang');

构造函数模式 ​

js
function Person(name) { //大写字母开头
    this.name = name;
}
var person = new Person("zhang");

以上方法的一个问题是：每个实例的方法在创建时都会跟着创建。可以将方法移到外面，但这样做对象失去封装性。

原型模式 ​

相关知识见[[#原型与原型链]]

js
function Animal() {}
Animal.prototype.name = 'animal';
var a1 = new Animal();

原型对象也可以采用字面值{}的方式赋值，但此时constructor属性不指向原函数，而且会切断现有原型与任何之前已经存在的对象实例之间的联系 ，它们引用的仍然是最初的原型。

组合模式 ​

构造函数模式用于定义实例属性，而原型模式用于定义方法和共享的属性。

动态原型模式 ​

在组合模式的基础上，把所有信息都封装在了构造函数中，在构造函数中初始化原型。

js
function Person(name) { //大写字母开头
    this.name = name;
	if (typeof this.sayName !== "function"){
		Animal.prototype.sayName = function(){};
	}
}
var person = new Person("zhang");

寄生构造函数模式 ​

工厂函数的再创建

js
function Sarray(name) {
	var o = new Array();
	o.push.apply(o, arguments);
	return o;
}
var o1 = new Sarray('zhang');

原型与原型链 ​

原型对象 ​

包含某个特定类型的所有实例共享的属性和方法。

  • 每一个被创建的函数有一个prototype属性（显示原型），这个属性指向原型对象。
  • 原型对象会获得一个constructor属性，这个属性是指向被创建函数。
  • 通过构造函数创建（new）的实例，包含一个属性__proto__ ([[Prototype]]，隐式原型)，指向构造函数的原型对象。

获取原型对象：

  • p.__proto__
  • p.constructor.prototype
  • Object.getPrototypeOf(p)

JS对象的关系：

  • 普通函数Foo及Object由Function创建。
  • 原型对象的__proto__指向Object.prototype（其指向null）。

ES6 可通过Object.setPrototypeOf()改变实例原型。

原型链 ​

通过让构造函数的prototype属性指向另一个类型的实例，此时的原型对象是实例，因此将包含一个指向另一个原型的指针，如此层层递进，就构成了实例与类型的链条。这就是原型链的基本概念。

函数也是对象。

javascript
function Foo(who) {
	this.me = who;
}
Foo.prototype.identify = function() {
	return "I am " + this.me;
};
function Bar(who) {
	Foo.call( this, who );
}
Bar.prototype = Object.create( Foo.prototype );
Bar.prototype.speak = function() {
	alert( "Hello, " + this.identify() + "." );
};
var b1 = new Bar( "b1" );
var b2 = new Bar( "b2" );
b1.speak();
b2.speak();

继承 ​

一般采用组合继承来实现：方法在原型上，而实例拥有自己的属性。

js
function SuperType(name){
  this.name = name;
  this.colors = ["red", "blue", "green"];
}
SuperType.prototype.sayName = function() { //原型方法
  console.log(this.name);
};
function SubType(name, age){
  SuperType.call(this, name); // 实例属性
}
SubType.prototype = new SuperType();
let instance1 = new SubType("Nicholas", 29);

显然，此种方法使得父类构造函数被调用两次。可以通过寄生模式省略掉new SuperType()时的初始化工作，称为寄生式组合模式。

以下可以将绑定继承，而不初始化父类构造函数：

js
function inheritPrototype(subType, superType) {
	let prototype = Object.create(superType.prototype);   
  	prototype.constructor = subType;                 
  	subType.prototype = prototype; // 因此subType instanceof superType.
}

类 ​

---[ES6]---

  • 类声明是基于已有自定义类型声明的语法糖，typeof值是"function"。
  • 没有变量提升；类内代码为严格模式，方法不可枚举。
  • 类名在类内为常量。
  • 支持可计算属性[]。
js
class A{
	constructor(name){
		this.name = name;
 	}
}
let a = new A('job');

等同于

js
let A = (function(){
	'use strict';
	const A = function(name){
		if(! this instanceof A){throw "error"}

	}

即时创建 先创建一个匿名类表达式，然后立即执行。依照这种模式可以使用类语法创建单例，并且不会在作用域中暴露类的引用。

js

	get name(){} // 访问器属性

继承 ​

内部使用寄生式组合模式实现继承。通过extends，可派生任意表达式，如果其可被解析为一个函数并且具有[[Construct]]属性和原型。

可继承静态成员。可继承内置对象，且原本在内建对象中返回实例自身的方法将自动返回派生类的实例。

js
class B extends A{
	constructor(){
		// 等价于 A.call(this)
		super(); 
	}
}

通过以下方式实现继承：

js
function _inherits(subClass, superClass){
	inheritPrototype(subClass, superClass); //前面有定义
	subClass.__proto__ = superClass; //使得静态属性可继承 
}

super使用

  • 只能在派生类构造函数和静态方法中使用。
  • 要么用它调用构造函数，要么用它引用静态方法。
  • 有构造函数一定要调用super()，且在访问this前，或者返回一个对象。

函数 ​

函数参数有自己的作用域和临时死区，其与函数体的作用域是各自独立的。

定义 ​

定义函数的方式有两种：函数声明，函数表达式。前者会有函数声明提升特点，后者是一种匿名函数赋值。

js
let a = function b(){}
//等价于, 因此全局中没有b变量 
let a = (()=>{
	let b = ()=>{}
	return b
})()

用途 ​

  • 通过new关键字调用函数，此时执行[[Construct]]函数，它创建一个通常被称作实例的新对象，将this绑定到实例上，再执行函数体。
  • 不通过new关键字调用函数，执行[[Call]]函数，直接执行函数体。

区分用途：通过call()还是new得到的实例。

js
function People(){
	if(this instanceof People){
 		console.log("new");
 	}
}

var p1 = new People(); // new
var p2 = People.call(p1); // new
var p3 = People(); //

----[ES6]---- new.target：元属性，函数内使用，判断函数是否通过new调用。若是其值为new操作符的目标，否则为undefined。

闭包 ​

模块 ​

提高代码复用率，方便进行代码管理。通常一个文件就是一个模块，有自己的作用域，只向外暴露特定的变量和函数。

模块化方式 ​

AMD ​

采用异步方式加载模块。所有依赖这个模块的语句，都定义在一个回调函数中。模块加载完成之后，运行该回调函数。

require.js 实现了该规范。define()定义模块，用require()加载模块。其核心原理是通过动态创建 script 脚本来异步引入模块，然后对每个脚本的 load 事件进行监听，如果每个脚本都加载完成了，再调用回调函数。

CMD ​

与AMD类似，区别在于：

  • 定义模块时：AMD 推崇依赖前置，提前声明其依赖的模块。CMD 推崇就近依赖，用到某个模块的时候再去 require。

  • 模块执行时：AMD加载后立即执行，模块执行顺序不确定。CMD加载完成后遇到 require 语句才执行，模块执行顺序与代码顺序一致。

sea.js实现了该规范。

js
/** AMD写法 **/
define(["a", "b"], function(a, b) { // 依赖a,b模块 
    a.doSomething();
    if (false) {
        // 即便没用到某个模块 b，但 b 还是提前执行了
        b.doSomething()
    }
});
/** CMD写法 **/
define(function(require, exports, module) {
    var a = require('./a'); //在需要时加载
    a.doSomething();
    if (false) {
        var b = require('./b');
        b.doSomething();
    }
});

CommonJS ​

基本用于服务端，如node。 运行时同步加载模块，以拷贝赋值的形式导入变量。

js
let a = 1;
module.exports = {a: a};
js
let mod = require("./test.js"); // 支持动态导入
console.log(mod.a);  // 1

ES6 ​

编译时记载异步加载模块，以只读引用的方式导出变量。

  • 静态导入：即在编译阶段确定导入的文件，被导入的文件仅会加载一次。
  • 导入的变量为只读绑定，不可变更。
  • import时模块代码被加载执行，因而可不绑定变量，用来执行某些全局操作。

自动运行在严格模式的JS代码，其顶部创建的变量不会被添加到全局共享作用域，且顶部this值为undefined。

export: 可以导入任意声明，导入变量时需用{}。

js
let b = 1;
export default let a = 12; // 默认值
export { b as B}; //重命名

import: 仅导入默认值时不需要{}；多个导入时，默认值需在非默认值前。

js
import a from "./test.js";
import {default as a, B as b} from "./test.js";
import * as c from "./test.js"; 整个模块导入

动态导入：import可用来动态的导入模块，其返回一个 promise。

js
let module = await import('/modules/my-module.js');

CommonJS 模块是对象， ES6 模块不是对象。

模块加载 ​

ES6定义了模块的语法，但没有定义如何加载这些模块。

浏览器加载 ​

默认情况下，浏览器同步加载 JavaScript 脚本，即渲染引擎遇到script>签就会停下来。

通过在标签中指定属性可以异步加载方式。 defer:：渲染完再执行。 async：下载完就执行。

html
<stript src="" defer></stript>
<stript src="" async></stript>

通过DOM动态创建的script元素默认以async方式加载。

也可以指定type类型加载ES6模块。默认以defer方式加载，也可重新指定加载方式。

html
<stript src="" type="module" async></stript>

规范 ​

  • 比较尽量使用===或 !==。
  • 不要在内置对象的原型上添加方法，如 Array, Date。
  • switch 语句必须带有 default 分支。
  • for、if必须使用大括号。
